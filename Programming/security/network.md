

Показывает таблицу маршрутизации.
```bash
netstat -nr
```
**Таблица маршрутизации** — это ==электронный «путеводитель» или набор правил, который сообщает вашему компьютеру (или роутеру), куда именно отправлять пакеты данных, чтобы они достигли цели==. 

Когда вы вводите адрес сайта или отправляете файл, система сверяется с этой таблицей, чтобы решить: отправить данные на роутер, в локальную сеть или обработать их внутри самого компьютера. 

Основные элементы таблицы

В выводе команды `netstat -nr` вы обычно видите следующие ключевые столбцы: 

- **Destination (Назначение):** Конечный адрес или целая сеть, куда вы хотите передать данные.
    - `default` — означает «весь остальной интернет». Если конкретного маршрута нет, данные идут сюда.
    - `127.0.0.1` — внутренняя «петля» (loopback) самого устройства.
- **Gateway (Шлюз):** «Посредник» или точка выхода. Если это IP вашего роутера (например, `192.168.0.1`), значит, данные для этого назначения нужно передать ему.
- **Netif (Interface):** Сетевой интерфейс, через который уйдут данные (например, `en0` для Wi-Fi в macOS).
- **Flags (Флаги):** Служебная информация о состоянии маршрута:
    - **U (Up):** Маршрут активен.
    - **G (Gateway):** Для этого пути требуется внешний шлюз (роутер).
    - **H (Host):** Маршрут ведет к конкретному узлу, а не к целой сети. 

Как это работает на практике

1. **Локальный трафик:** Если вы обращаетесь к другому компьютеру в вашей же Wi-Fi сети, таблица видит, что это «своя» подсеть, и отправляет данные напрямую через интерфейс `en0`.
2. **Внешний трафик:** Если вы открываете `google.com`, таблица не находит точного совпадения для этого IP и использует маршрут `default`, отправляя запрос на шлюз (ваш домашний роутер).
---

получить свой ip
- **Для Wi-Fi:** `ipconfig getifaddr en0`.
- **Для Ethernet:** `ipconfig getifaddr en1`.
- **Универсальный фильтр:** `ifconfig | grep "inet "`
