#### Чем отличается паттерн Facade от фасадов Laravel?
Laravel-фасады используют Dependency Injection с возможностью статического вызова методов

Фасады в Laravel вдохновлены классическим паттерном Facade, но имеют некоторые особенности:
1. **Реализация**:
    - В классическом `Facade` объект создаётся и управляется явно пользователем.
    - В **фасадах Laravel** (таких как `Facades\Auth`, `Facades\Mail`) используется скрытый доступ к объекту через сервисы контейнера (Dependency Injection).

2. **Механизм работы**:
    - Фасады Laravel предоставляют статический интерфейс, но под капотом каждый вызов перенаправляется через объектную модель с использованием **сервис-контейнера**.
    - Это достигается через магию PHP (`__callStatic`).

3. **Примеры Laravel фасадов**:
    - Фасад `Auth` предоставляет простую точку входа к службам аутентификации.
    - Фасад `Storage` предоставляет интерфейс к файловой системе.
    - Фасад `Mail` используется для отправки почты.

Пример Laravel-фасада:

``` php
use Illuminate\Support\Facades\Storage;

// Доступ к файловой системе через фасад
Storage::put('example.txt', 'Hello World');
$content = Storage::get('example.txt');
```

#### Примеры использования фасадов Laravel
Фасады в Laravel позволяют упрощать работу с сервисами:
1. **Auth::attempt()**
    - Фасад скрывает реализацию системы аутентификации (выбор хранилища пользователей, шифрование паролей и т.д.).

2. **Route::get()**
    - Фасад предоставляет доступ к подсистеме роутинга.

3. **Cache::remember()**
    - Фасад для взаимодействия с кешем – Redis, Memcached и т.д.

Laravel-фасады одновременно сочетают в себе идеологию паттерна Facade для упрощения интерфейса и возможности Dependency Injection для большей гибкости.

#### Отличие Facade от Dependency Injection
1. **Facade**:
    - Складывает в одном простом интерфейсе целую подсистему.
    - Пример: Фасад в Laravel предоставит метод `Mail::to()` и скроет все процессы отправки писем.

2. **DI (Dependency Injection)**:
    - Предоставляет объект (или класс) через конструктор или метод, чтобы напрямую с ним работать.
    - Пример: Мы можем внедрить класс-обработчик `Mailer` в контроллер.
