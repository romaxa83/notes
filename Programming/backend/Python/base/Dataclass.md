Класс данных (`dataclass`) — это тип класса, который в основном используется для хранения данных и автоматически генерирует шаблонный код, такой как конструкторы (`__init__`) и методы сравнения (`__eq__`). В Python для этого используется декоратор `@dataclass` из модуля `dataclasses`, что позволяет создавать компактные и легковесные классы, предназначенные для хранения данных, а не для выполнения сложной логики. 

Основные характеристики

- **Автоматическая генерация методов:** Декоратор `@dataclass` автоматически создает методы, такие как `__init__`, `__repr__` (для строкового представления) и `__eq__` (для сравнения объектов), избавляя разработчика от ручного написания boilerplate-кода.
- **Использование аннотаций типов:** Определение полей класса с помощью аннотаций типов (например, `name: str`) позволяет декоратору генерировать необходимый код.
- **Простота и читаемость:** `dataclass` упрощает написание кода и делает его более читаемым, поскольку классы остаются очень простыми и сфокусированными на хранении данных.
- **Неизменяемость (опционально):** Для создания неизменяемых объектов, которые не могут быть изменены после создания, можно использовать аргумент `frozen=True` в декораторе `@dataclass`.
- **Доступ к данным:** По умолчанию классы данных имеют геттеры и сеттеры, но их можно сделать неизменяемыми для предотвращения случайных ошибок при работе с данными. 

Пример использования в Python

python

```python
from dataclasses import dataclass

@dataclass
class User:
    name: str
    age: int = 0 # Значение по умолчанию
    is_active: bool = True

# Создание экземпляра класса
user = User("Alice")

# Печать объекта (автоматически сгенерированный __repr__)
print(user) # Выведет: User(name='Alice', age=0, is_active=True)

# Сравнение объектов (автоматически сгенерированный __eq__)
user1 = User("Bob", 25)
user2 = User("Bob", 25)
print(user1 == user2) # Выведет: True
```

Когда стоит использовать

- Для создания структур данных, таких как "объекты-значения" (value objects) или DTO (Data Transfer Objects).
- Когда класс нужен в первую очередь для хранения данных, а не для выполнения сложной бизнес-логики.
- Для создания неизменяемых объектов, обеспечивающих безопасность данных.
